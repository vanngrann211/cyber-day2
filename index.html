Today you open a terminal for the first time and immediately put it to work. OverTheWire Bandit is a hacking game played entirely through the command line. Each level hides a password. Concept

What Is a Terminal?

A terminal is a text-based way to control a computer. Instead of clicking icons, you type commands. It looks plain. It is the most powerful tool on any computer.

WHY HACKERS LIVE HERE

Every server on the internet runs Linux with no icons and no mouse. To work with real systems you must use the terminal. Bandit teaches you the terminal by making a game out of it.

Setup

Open Your Terminal

WINDOWS

Press the Windows key. Type: powershell. Press Enter. A blue or black window opens. PowerShell can run the ssh command you need today.

MAC

Press Command + Space. Type: Terminal. Press Enter. A window opens. This is your terminal.

YOU WILL SEE

A prompt with a blinking cursor waiting for you to type. Nothing is broken. It is ready for a command.

Command Primer

The Commands You Will Use Today

Bandit runs on Linux, so everyone uses the same Linux commands once connected. Here are the essentials. Do not memorize them, just know they exist. You will learn each one as you need it.

ls list the files here cd foldername go into a folder cd .. go back one folder cat filename read a file file filename find out what kind of file it is find search for files grep "word" file search inside a file for a word ls -a list ALL files including hidden ones
KEEP THIS HANDY

Use your printed cheat sheet for the full reference. Each Bandit level below tells you which of these to reach for.

Concept

What Is SSH?

SSH stands for Secure Shell. It lets you log into another computer over the internet and use its terminal as if you were sitting in front of it. Bandit lives on a server you reach with SSH.

WHAT IT GIVES YOU

A terminal session on the Bandit server. You type commands, they run there, results come back to you.

ALREADY INSTALLED

SSH is built into Windows PowerShell and Mac Terminal. Nothing to download.

Concept

The SSH Command

Every SSH connection follows the same pattern:

ssh username@hostname -p port
  • username - who you log in as (bandit0, bandit1, and so on)
  • @hostname - the address of the server
  • -p port - which door to knock on (Bandit uses 2220)
FIRST TIME WARNING

The first time you connect, it asks "are you sure you want to continue connecting?" Type yes and press Enter. Normal, happens once per server.

How Bandit Works

The Rules of the Game

  • Each level is a separate user account on the Bandit server
  • You SSH in using that level's username and password
  • Somewhere on that account is a file with the password for the NEXT level
  • Find the password, use it to SSH into the next level, repeat
  • Write every password down. If you lose one, you redo that level
YOUR MOST IMPORTANT HABIT

Open Notepad or TextEdit right now. Keep it open all day. Paste every password the moment you find it.

Level 0

Connect to the Server

Level 0 is the connection itself. In your terminal, type:

ssh bandit0@bandit.labs.overthewire.org -p 2220
  1. Type the command above and press Enter
  2. When asked to continue, type yes and press Enter
  3. Password: bandit0
  4. You are now inside the Bandit server
  5. Type ls to see the files here
  6. Type cat readme to read the file and get Level 1's password
  7. Paste that password into your text file immediately
Advancing

How to Move to the Next Level

Once you find a password, disconnect and reconnect as the next user:

exit disconnect from current level ssh bandit1@bandit.labs.overthewire.org -p 2220 connect as level 1 (change the number each time)
THE PATTERN

bandit0 finds the password, then you log in as bandit1 with it. bandit1 finds the next, you log in as bandit2. The username number always matches the level you are entering.

Level 1

Challenge 1: The Dash File

1
Reading a file named with a dash
SKILL: special filename trick
The password is in a file called - (just a dash). The problem: cat - does not work, because the dash makes the terminal wait for keyboard input instead of reading the file. You need a workaround.
Command Hints

Read the file by giving its location, not just its name. The prefix ./ means "this file right here". Combine with cat. So you want cat then ./ then the dash.

Level 2

Challenge 2: Spaces in a Filename

2
Filename with spaces
SKILL: quoting filenames
The password is in a file whose name has spaces in it. The terminal normally treats each space as a separator, so typing the name plainly confuses it.
Command Hints

Wrap the whole filename in quotes: cat "...". Or put a backslash before each space. Easiest of all: type the first few letters and press TAB to auto-complete.

Level 3

Challenge 3: The Hidden File

3
Hidden file inside a folder
SKILL: revealing hidden files
The password is in a hidden file inside a directory called inhere. A normal ls will not show it. Hidden files on Linux start with a dot.
Command Hints

Go into the folder with cd inhere. List everything including hidden files with ls -a. Then read the hidden file with cat.

Level 4

Challenge 4: Find the Readable File

4
One readable file among many
SKILL: the file command
The inhere directory has many files. Only one holds readable text. The rest are binary data that print as garbage. You need to identify the human-readable one without opening each.
Command Hints

The file command tells you what kind of data a file holds. Run file ./* to check all of them at once. Look for the one that says ASCII text, then cat it.

Level 5

Challenge 5: Find by Properties

5
Find a file matching specific traits
SKILL: find with filters
The password file is buried in many subfolders inside inhere. It has three traits: human-readable, exactly 1033 bytes, and not executable.
Command Hints

Use find with filters. -size 1033c matches exactly 1033 bytes. -not -executable rules out programs. Start the search from inhere and let find dig through every subfolder.

Level 6

Challenge 6: Search the Whole Server

6
File hidden somewhere on the system
SKILL: find by owner and size
The file is somewhere on the entire server. It is owned by user bandit7, group bandit6, and is 33 bytes. Searching everything produces many permission errors you can ignore.
Command Hints

Search from the top with find /. Add -user bandit7, -group bandit6, and -size 33c. Hide the errors by adding 2>/dev/null at the end.

New Idea

2>/dev/null: Hiding Errors

Level 6 introduces a trick that hides error messages so they do not bury your results.

find / -size 33c 2>/dev/null
  • 2 means error output
  • > means send it somewhere
  • /dev/null is a black hole that throws away anything sent to it
  • Result: errors vanish, real results still show
USED CONSTANTLY

Searching a whole system always produces permission-denied noise. This trick is one of the most-used in real security work.

Level 7

Challenge 7: Word in a Big File

7
Password next to a specific word
SKILL: grep
The password sits in data.txt right next to the word millionth. The file has thousands of lines. Reading it by hand would take forever.
Command Hints

This is what grep is built for. Search for the word millionth inside data.txt. grep prints only the line that contains it, which holds the password.

Level 8

Challenge 8: The Line That Appears Once

8
Unique line among duplicates
SKILL: sort and uniq
data.txt has many lines and most repeat several times. The password is the only line that appears exactly once. Too many lines to spot by eye.
Command Hints

Chain two commands with a pipe. First sort data.txt groups identical lines together. Then pipe into uniq -u, which keeps only lines that never repeat.

New Idea

Pipes: Chaining Commands

Level 8 uses a pipe for the first time. A pipe takes the output of one command and feeds it straight into the next.

sort data.txt | uniq -u
  • sort data.txt sorts every line
  • | the pipe passes that result forward
  • uniq -u keeps only the unique lines
THE BIG IDEA

Pipes let you build pipelines: each command does one job, chained together to solve big problems. Professional terminal work uses pipes constantly.

Level 9

Challenge 9: Strings in a Binary File

9
Readable text buried in binary data
SKILL: strings command
data.txt is mostly binary data with a few readable pieces mixed in. The password is one of those readable strings, marked by several equals signs in front of it. Reading the file directly shows gibberish.
Command Hints

The strings command pulls all readable text out of any file. Run strings on data.txt, then pipe to grep = to filter for the lines with equals signs.

Level 10

Challenge 10: Base64 Decoding

10
Decode a scrambled file
SKILL: base64 decoding
data.txt holds text encoded in base64, a way of scrambling text into random-looking letters and numbers, usually ending in equals signs. You need to decode it back to normal.
Command Hints

Look at the file with cat data.txt first. Then decode it: base64 -d data.txt. The -d flag means decode. The output is a readable line containing the password.

Commands You Learned

Your New Toolkit

  • ssh user@host -p port - connect to a remote server
  • ls, cd, cat - list, navigate, and read
  • ls -a - show hidden files
  • ./filename - reference a file in the current folder
  • file - identify what a file contains
  • find with -size, -user, -group - search by traits
  • grep - search inside files
  • sort, uniq, strings, base64 - process and decode data
  • pipes and 2>/dev/null - chain commands and hide errors
Persistence

Getting Stuck Is Normal

WHEN YOU CANNOT FIGURE IT OUT

Read the challenge again slowly. Look at the command hints on the slide. Try the command. Get it wrong. Read the error. Adjust. Try again. This struggle is the lesson. Every level you break through on your own builds real skill.

LAST RESORT

If you are truly stuck after real effort, the Bandit site at overthewire.org/wargames/bandit has full explanations. Use them only after you have genuinely tried.
